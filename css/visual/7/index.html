7. Display

Display property controls how elements are laid out on a page. Values include block, inline, inline-block and none.

7.1. Inline

Elements are laid out side by side
Default height and width is based on content
Content and element wraps to the next line in overflow scenario
Default for text elements like span, i, b, input etc

Text 1 Text 2 Text 3
Content and elemnt wraping to next line

Center text

Since width of inline element is determined directly by the content within it, it is not possible or needed to center text within it

7.2. Block

Block elements are laid out one after another
Default width spans the entire width of the page
Only content wraps to the next line in overflow scenario
Default for elements like div, p, table etc

Div 1
Div 2
Overflow div, element doesn't wrap. Height of element increases if no default height specified

Center text

Text-align center can be used to center text

Text is centered

7.3. Inline-block

Inline-block elements are laid side-by-side.
Top-bottom margins and height-width are respected
Useful for laying out nav bars etc.

Span 1 displayed as inline block Span 2 displayed as inline block
Div displayed as inline block

7.4. None

Completely removes the element from flow

Visibility

It is possible to hide an element by setting visibility: hidden. Element is part of the flow but hidden.

7.5. Float and Clear

Float places elements on left or right side of the container, allowing other elements to wrap around them. Applies to both inline and block elements.

The clear property specifies whether an element must be moved below floating elements that precede it.
