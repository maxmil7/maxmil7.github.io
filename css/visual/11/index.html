11. Grid

dislpay: grid is used to divide a page into multiple regions based on rows and columns (known as tracks)
Grid properties define track size, gaps, alignment, flow and mapping of items within tracks.
Properties on grid items define order, span and alignment.
Grid is automatically extended based on grid content.

11.1. Grid container properties

  • grid-template-columns: definies column size within the grid
  • grid-template-rows: defines row size within grid
  • grid-auto-columns: defines column size for implicit (automatic) columns
  • grid-auto-rows: defines row size for implicit (automatic) rows
  • grid-template-area: specifies named grid areas. Can be later associated with grid items
  • grid-template: shorthand for grid-template-area, grid-template-rows, grid-template-columns
  • grid: same as grid-template but prevents values from cascading in separately
  • grid-auto-flow: specifies how items get flowed in the grid (row, column, dense or their combination)
  • row-gap, column-gap, grid-gap: sets gaps between rows and columns. Shorthand for row-gap and column-gap.
  • align-items: controls item alignment along the block (column) axis. Axis doesn't not switch like flexbox
  • justify-items: controls item alignment along the inline (row) axis
  • place-items: shorthand for align-items and justify-items
  • align-content: controls track alignment along block axis
  • justify-content: controls track alignment along inline axis

11.2. Grid item properties

  • grid-row-start, grid-row-end: specifies item row start and end position location
  • grid-row: shorthand for row-start/row-end
  • grid-column-start, grid-column-end: specifies item column start and column end location
  • grid-column: shorthand for column-start/column-end
  • grid-area: shorthand for row-start/column-start/row-end/column-end or named area
  • order: specifies item placement order
  • align-self: specifies item location along the block axis (single)
  • justify-self: specifies item location along the inline axis

11.3. Glossary

  • tracks: space between two grid lines. Can be row track or column track.
  • naming grid lines: grid lines can be named when defining tracks (using grid-template-rows, grid-template-columns or grid-template)
    If areas are defined, areaname-start or areaname-end can be used as grid line names. Names can then be used when defining item areas.
  • fractional: represents fraction of space available. Useful when defining track sizes
  • repeat(): function representing a repeated fragment of the track list
  • minmax(): function defining a size >= min and >= max
  • auto-fill, auto-fit: can be used for automatically creating columns eg.(repeat(auto-fit, minmax(150px, 1fr));
  • fit-content: function used to define row/column width based on content Eg. fit-content(max value).
  • span: can be used to define item span along row or column
